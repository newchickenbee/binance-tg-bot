### 1. 任务方案配置-架构扩展与策略模块
- [ ] 模块新增：「任务方案-配置策略」页面中新增「非目标受邀人奖励策略」专属模块设置。
- [ ] 多策略层级支持：支持动态添加多个奖励策略子项，限制最多可添加 5 个。
- [ ] 命中优先级控制：实现策略排序机制，强制定义基于自上而下的逐层兜底匹配逻辑。

### 2. 策略字段扩展及入库逻辑开发
- [ ] **奖励人群功能**：
    - [ ] 实现与 CDP 客群系统接口的数据防腐层对接及请求下拉。
    - [ ] 支持输入、查询和关联已上线的外部 CDP 客群。
    - [ ] 特定逻辑：未选择人群时，系统判定受众覆盖“全部非目标受邀人”。
- [ ] **奖励类型**：
    - [ ] 枚举限定控制：固定值强制为“优惠券”。
- [ ] **奖励方案**：
    - [ ] 实现方案联想搜索。
    - [ ] 联动展示和支持下拉选择“优惠券方案”和“优惠券券池方案”。
- [ ] **频次机制定义**：
    - [ ] 频次逻辑：每人奖励次数写死，限定配置且落库默认单人 1 次上限。
- [ ] **规则设定功能**：
    - [ ] 领取规则限定：锁定值为“系统发放”选项。
    - [ ] 使用规则限定：锁定值为“持有”选项。
- [ ] **奖励描述储存**：
    - [ ] 支持输入前端提示文案并验证最大字符长度合法性。

### 3. 「配置策略」编辑页/新建页视图层与交互优化
- [ ] 核心配置区排版：「适用范围」下方新增渲染「受邀人」配置版块。
- [ ] 「目标受邀人」字段配置：
    - [ ] 支持选择大类：“新人/非新人”枚举。
    - [ ] 事件联动：若圈定“非新人”，动态展露 CDP 客群输入框，并支持通过接口搜索和选择已上线CDP分群。
- [ ] 「非目标受邀人奖励」核心开关开发：
    - [ ] 新增布尔类型启用字段（枚举展示：是/否，默认状态：否）。
    - [ ] 组件联动控制：开启“是”时，解锁高亮「添加策略」按钮；为“否”时按钮置灰且禁用策略项新增功能。

### 4. 表单与列表原子化操作设计 (CRUD)
- [ ] 列表子策略项动作支持组：
    - [ ] 实装策略记录的「删除」、「复制」操作以及所需的新增记录克隆逻辑。
    - [ ] 实装序列更替控制：「上移」、「下移」、「置顶」、「置底」四种层级控制方法以及位置数据库重排序更新。
- [ ] 按活动工作流方案状态严格校验检查（State Guards）：
    - [ ] 按照PRD要求的方案流转化条件（已新建、未开始、已开始、已结束等），针对表单原子化按钮动态控制功能的可触及性。
    - [ ] 后端表单校验拦截开发：验证保存请求提交时，当前各非目标受邀人策略客群所配置的 CDP 客群对象与总“受邀人客群”的数组实体是否存在逻辑交叉。倘若存在重叠对象集，则阻断落库动作并上抛客户端异常：“人群分群重合，请修改”。

### 5. 执行引擎 - 命中路由下发与发券链路
- [ ] 核心匹配算法重构：
    - [ ] 根据落库数据的层级权重，迭代现存发券中间件规则，保证基于页面序列从上往下逐层校验人群命中匹配。
    - [ ] 兜底发券处理：同一个受邀用户可能符合子策略网的交叉规则，需在此处实现强排他计算，使其仅且只命中优先级最高的一个顺位策略，断绝出现连带重叠违规发券的情况。
